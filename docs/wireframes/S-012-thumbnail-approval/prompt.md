# S-012 サムネ承認（バルク + 単冊） — ChatGPT ワイヤーフレーム生成プロンプト

## 画面情報
- 画面 ID: S-012
- 画面名: サムネ承認（バルク + 単冊）
- 対応機能 ID: F-006, F-007, F-014, F-019, F-049
- 元設計書: `docs/04-ui-design.md` §4 S-012
- 想定画像:
  - `desktop.png` — バルクグリッドモード
  - `single.png` — 単冊詳細モード（モード切替後の画面）
  - `mobile.png` — モバイル 1 カラム
  - `empty.png` — 承認待ちなし
  - `error.png` — 画像生成失敗

## ChatGPT 使い方
1. ChatGPT (GPT-4o / 画像生成有効) を開く
2. 共通プロンプト + 各バリアントを結合してコピー
3. ChatGPT に貼り付け → 画像生成
4. 出力 PNG を本ディレクトリに保存

## 共通プロンプト（全画像で先頭に置く）

```
You are a senior UX designer creating a low-fidelity wireframe for a Japanese web application called "A2P" (Amazon KDP publishing automation tool, single-operator use).

This is a BULK OPERATION + COMMENT-CENTRIC screen:
- 各書籍カード左上にチェックボックス、ヘッダーに全選択
- 画面下部に BulkActionBar 固定: "N 件選択中 / [選択候補を一括採用] [全候補を再生成] [カバーテキスト再生成] [選択解除]"
- 件数バッジを目立たせる
- 画像の上に座標領域指定型コメント（点線枠の例示 + 中に `[+]` バッジ）
- 既存コメントは画像右側にバッジ列、優先度色 (must=赤 / should=黄 / may=青)

Style rules:
- Pure black-and-white, light gray for de-emphasis.
- カバー画像はプレースホルダ枠（縦長 3:4 比率、内部に "{表紙画像}" のテキスト + 帯状のタイトルプレースホルダ）
- No rounded corners or shadows. Japanese labels.

Persistent UI:
- Header (64px): "A2P" + グローバル検索 + CostMeter "3.2万/5万" + AlertBadge "3" + CommentBadge "12 (must:4)" + 設定 + ユーザーメニュー
- Sidebar (240px): ホーム / 出版パイプライン（テーマ候補 / 新規プロジェクト・バッチ計画 / アウトライン承認 / サムネ承認 / KDP 入稿） / 書籍（書籍ライブラリ / 修正コメント） / 分析（売上・KPI / コスト詳細） / モデル & プロンプト（モデル割当 / モデルカタログ / A/B 比較 / プロンプト管理 / 改訂承認） / 運用（ジョブログ / アラート / KDP 自動入稿 / 監査ログ / アカウント管理 / 設定）
- Sidebar 下部: JobTicker "実行中 3 / 上限 5"
```

## 設計意図メモ（ChatGPT には渡さない）

- バルクモードと単冊モードを 1 画面でモード切替する設計（業務要件「夜セットの一括承認 + じっくり見たい時の単冊比較」）。
- 座標領域コメントは F-049 受け入れ基準「サムネに領域指定でコメント」を視覚化。
- ジャンル別意匠原則を右サイドに常設し、カバーテキスト判断の基準を運用者に提示。
- BulkActionBar の「カバーテキスト再生成」は F-006 と F-007 が分離していることを反映。
