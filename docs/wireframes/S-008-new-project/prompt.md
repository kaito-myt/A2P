## desktop.png プロンプト

```
Layout: 1440x900 pixels, desktop browser view.

[Header / Sidebar は共通プロンプトの通り。アクティブナビは "新規プロジェクト/バッチ計画"]

Main content area (2 カラム: 左 65%、右 35%):

### Section 1: ページヘッダー（全幅）
- パンくず: "ホーム > 出版パイプライン > 新規プロジェクト/バッチ計画"
- タイトル: "新規プロジェクト / 夜間バッチ計画"
- 右側: `[ バッチ保存 ]` `[ 即時キック ]`（プライマリ）

### Section 2: 採用済みテーマ選択リスト（左カラム上）
- セクション見出し "採用済みテーマ (5 件選択中)" + `[ + テーマ追加 ]`（→ S-006）
- 各行: チェックボックス + テーマタイトル + ジャンル + 想定読者 + `[×]` 削除
- 5 行表示（S-006 から引き継いだ採用済みテーマ）

### Section 3: バッチ設定フォーム（左カラム中段）
- セクション見出し "バッチ設定"
- 開始時刻: 入力 `[ 2026-05-21 23:00 ]` (datetime picker 風)
- 並列度: ラジオ `( ) 1  ( ) 2  ( ) 3  ( ● ) 4  ( ) 5`
- 希望完了時刻: 入力 `[ 2026-05-22 07:00 ]`
- 注記: "並列度 4 / 上限 5。Phase 1 既定"

### Section 4: モデル割当プレビュー（左カラム下、S-019 のサブセット埋め込み）
- セクション見出し "モデル割当プレビュー" + `[ 詳細編集 ]`（→ S-019）
- 小型マトリクス（縦: 役割 6 つ / 横: ジャンル 3 + デフォルト）
- 各セル: "Claude Opus 4.7" のような短縮表記
- 下に「この実行だけ上書き」トグル + 上書きフォーム（折りたたみ）

### Section 5: 予測コスト・予測完了時刻（右カラム上、目立たせる）
- 大カード "予測サマリ"
- 縦並び:
  - 採用テーマ数: 5 冊
  - 予測総コスト: ¥1,840
  - 1 冊平均: ¥368
  - 予測完了時刻: "2026-05-22 06:42"
  - 月次残高との突き合わせ: "残額 ¥17,850 → 実行後 ¥16,010" (進捗バー)
- 警告レベルバッジ: "🟢 安全圏" or "🟡 注意" 等（色なしテキスト）

### Section 6: バリデーション結果（右カラム下）
- セクション見出し "事前チェック"
- チェック項目リスト:
  - ✓ 採用テーマ数 ≦ 並列度許容
  - ✓ 月次予算超過予測なし
  - ✓ 全役割にモデル割当あり
  - ✓ アカウントの長期プランと整合
- 全項目 OK 状態
- 1 つでも NG なら赤バッジ + 修正リンク

### Section 7: 強制続行スイッチ（右下、小さく）
- 通常は折りたたみ
- 月次予算超過予測時にのみ展開、`[ 月次予算上限を超えても続行 ]` トグル
```

- UC-01「一晩 5 冊バッチ」の中核画面。1 画面で「採用テーマ → 並列度 → モデル → 予測コスト → キック」を完結。
